Orientational Challenges

How to present the content or the available navigation options to the users before taking actions?

4 Patterns

A conversation must provide a motivated, recognizable high-level structure that prevents recalling the entire architecture and convey the main functions.

The first baseline challenge is enabling the user to apply in-the-large navigation mechanisms and to create a web space concept map. A related pattern is view in the large, a solution achievable when a conversation provides a motivated, recognizable high-level structure that prevents recalling the entire architecture and adequately conveying the main website’s functions.

Useful tip:
Respect vertical structure parallelism providing the user with same-type of content on the same-depth of different sections.

EXAMPLE CONVERSATION

A conversation must present macro-areas clustering according to logical and semantic criteria, to convey notions of context.

The user challenge of delivering a meaningful and logical content clusterization can be resolved through the employment of macro-areas clustering according to logical and semantic criteria. Content grouping and nesting also conveys notions of context.
Moreover, supplying the users with a motivated hierarchy prevents them from recalling the entire architecture, because nodes are logically linked together.
This ideally leads the user to further adopt an intuitive navigational behaviour. The arrangement of correlation between different nodes, associated with the pattern link predictability, allows the users to easily orient themselves, proceeding in broadening their search.

Useful tip:
Add useful alternatives: If the expected content is not available in a specific section, provide an immediate, landmarked path alternative to reach it. If possible, intervene on website information structure (modify the HTML element’s semantic tags) to apply Proper labelling according to logical criteria.

Conversational Blocks

EXAMPLE CONVERSATION

A conversation must support direct probing about the offerings in natural language, directly pose questions to ask whether the website is supplying a specific feature

With the aim of directly inquiring the conversational agent about punctual, fast-served requests, the navigation by question and answer is developed as a suitable solution. A conversation should support direct probing about the offerings in natural language, and posing questions to ask whether the website is supplying a specific feature.

Knowing the complete Website offer, prior to formulating a request, constitutes a big obstacle in navigation. Explicitly formulating state requests even without having fully explored or learnt the whole structure, grants the user the possibility to fulfil specific information needs with a more direct access to the offerings or features. With this mechanism the user can directly pose questions to the bot to ask whether the website is offering a specific feature or narrow down the list of offerings. In this case, nodes are matched against the natural language expression of the information needed.

Useful tip:
Allow the user to search “locally” (for example, at single-page level) as well as on global level (the entire website). This mechanism can be fed even by an external navigation context, for example a question when coming from a search engine.

Content Fruition Challenges

How to improve the fruition of textual content?

3 Patterns

A dialogue must convey content's preview relied on the information extracted from a node or through abstract summarisation.

Addressing the need of preventing the user from unsought and unneeded explorations, the concept of supply with a conversational skimming alternative for non visual mediums:
a dialogue should convey content’s preview relied on the information extracted from a node or through abstract summarisation. The approach can rely on the information extraction (employ cut original content) from a node or abstraction summarisation (shorten paraphrased content).

Useful tip:
Enable personalised content access to allow the user to both interact with summarised and full-version information.

Conversational Blocks

EXAMPLE CONVERSATION

A conversation must provide access to a segmented content, whose interaction is fully administrable by users.

With the aim of supplying to users with content subdivision for an easy-reading and avoiding a cognitively demanding interaction, a conversation should provide access to a segmented content, whose interaction is fully administrable by users. This pattern therefore proposes to relieve from the effort in maintaining a long-term attention on the content read by the conversational agent. The solution can be introduced by dividing content from the same node into sections and giving the user the possibility to fully control the interaction (interrupt and resume after a pause).

Useful tip:
Add intuitive back and forth commands to easily manage the content-reader feature among multiple content sections. Moreover, ensure intent-recognizable commands’ analogies in natural language.

Scaffolding Challenges

Which are commands that have to be always available during navigation?

3 Patterns

A conversation must enable localization cues during the page browsing, quickly grasping the system feedback.

To overcome the challenge of lacking multiple-levels feedback on the system status and to reduce the time needed to interpret the user input, a conversation should enable localization cues during the page browsing, quickly grasping the system feedback (e.g. displacement from a node, to another one from link, or the page is loading). Landmarks not necessarily displaying content.
Signals describing current system status could be provided employing a recurrent statement or a specific earcon.

Useful tip:
Add user selection checking to prevent errors or even unintentional actions. Probe the user for a further confirmation if the activity involves significant changes in explorational flows.
